Pengertian Polimorfisme dan Abstraksi

Abstraksi dan polimorfisme adalah konsep OOP dimana abstraksi menyembunyikan detail yang rumit dan hanya menampilkan bagian penting, sedangkan polimorfisme memungkinkan satu method memiliki banyak bentuk atau perilaku berbeda tergantung objeknya.

🔹 Pewarisan

Pewarisan (Inheritance) adalah konsep dalam OOP yang memungkinkan sebuah class (child/anak) mewarisi sifat dan method dari class lain (parent/induk), sehingga tidak perlu membuat ulang kode yang sama.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A class Hewan { void bersuara() => print('...'); } class Kucing extends Hewan { @override void bersuara() => print('Meong'); } void main() { Kucing k = Kucing(); k.bersuara(); }

🔹 Pengantar Polimorfisme

Polimorfisme adalah konsep dalam OOP yang berarti “banyak bentuk”, yaitu kemampuan suatu objek untuk memiliki perilaku yang berbeda meskipun menggunakan method yang sama. Dalam polimorfisme, objek dari class turunan (subclass) bisa diperlakukan sebagai class induk (superclass), tetapi tetap menjalankan fungsi sesuai jenis objeknya.

🔹 Polimorfisme Inheritance

Variabel bertipe parent bisa menyimpan objek child dan Method yang dipanggil mengikuti objek aslinya.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A class Bentuk { void gambar() => print('Menggambar bentuk'); } class Lingkaran extends Bentuk { @override void gambar() => print('Menggambar lingkaran'); } class Persegi extends Bentuk { @override void gambar() => print('Menggambar persegi'); } void main() { Bentuk b1 = Lingkaran(); Bentuk b2 = Persegi(); b1.gambar(); b2.gambar(); }

🔹 Manfaat Polimorsime

Fleksibel Mudah dikembangkan Bisa digunakan kembali (reusable) Mudah dirawat (maintainable)

🔹 Praktik 1: Polimorfisme dengan List

Gunakan List Isi dengan berbagai objek child Loop dan jalankan method
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A class Hewan { void bersuara() => print('Hewan bersuara'); } class Kucing extends Hewan { @override void bersuara() => print('Meong'); } class Anjing extends Hewan { @override void bersuara() => print('Guk guk'); } class Burung extends Hewan { @override void bersuara() => print('Cuit cuit'); } void main() { List daftarHewan = [ Kucing(), Anjing(), Burung(), ]; for (var h in daftarHewan) { h.bersuara(); } }

🔹 Pengantar Abstraksi

Menyembunyikan detail dan hanya menampilkan fungsi penting.

🔹 Abstract Class

Tidak bisa dibuat objek langsung Bisa punya: Method abstract (tanpa isi) Method biasa (dengan isi)
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class Kendaraan { String? merk; void start(); // abstract void stop() { print('Kendaraan berhenti'); } } class Mobil extends Kendaraan { @override void start() { print('Mobil $merk menyala'); } } void main() { Mobil m = Mobil(); m.merk = "Toyota"; m.start(); m.stop(); }

🔹 Abstract Methoad

Abstract method adalah method yang tidak punya isi dan harus diisi oleh class turunan.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class AlatMusik { void mainkan(); void info() { print('Ini adalah alat musik'); } } class Gitar extends AlatMusik { @override void mainkan() { print('Jreng jreng...'); } } void main() { Gitar g = Gitar(); g.info(); g.mainkan(); }

🔹 Interface di Dark

Interface di Dart adalah kontrak/aturan yang berisi method yang harus diimplementasikan oleh class lain.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A interface class Flyable { void terbang() => print('Terbang...'); } class Bird implements Flyable { @override void terbang() => print('Burung terbang tinggi'); } class Plane implements Flyable { @override void terbang() => print('Pesawat terbang cepat'); } void main() { Bird b = Bird(); Plane p = Plane(); b.terbang(); p.terbang(); }

🔹 Static Member

Milik class, bukan objek Dipanggil langsung dari class
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A class MathUtils { static double pi = 3.14; static int add(int x, int y) { return x + y; } } void main() { print(MathUtils.pi); print(MathUtils.add(10, 20)); }

🔹 Praktik 2: Sistem Pembayaran

Sistem pembayaran adalah penerapan konsep abstraksi dan polimorfisme dalam OOP, dimana dibuat satu class sebagai dasar (template) dengan method `bayar()`, lalu berbagai jenis pembayaran seperti kartu kredit, e-wallet, dan tunai mengimplementasikan method tersebut dengan cara yang berbeda. Dengan begitu, satu method yang sama dapat digunakan untuk berbagai objek dengan perilaku yang berbeda.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class MetodePembayaran { void bayar(double jumlah); } class KartuKredit implements MetodePembayaran { @override void bayar(double jumlah) { print('Bayar $jumlah dengan Kartu Kredit'); } } class EWallet implements MetodePembayaran { @override void bayar(double jumlah) { print('Bayar $jumlah dengan E-Wallet'); } } class Tunai implements MetodePembayaran { @override void bayar(double jumlah) { print('Bayar $jumlah dengan Tunai'); } } void main() { List metode = [ KartuKredit(), EWallet(), Tunai() ]; for (var m in metode) { m.bayar(100000); } }

🔹 Studi Kasus: Sistem Bentuk Geometri

Sistem bentuk geometri adalah contoh penerapan abstraksi dan polimorfisme dalam OOP, dimana dibuat satu class dasar (misalnya Bentuk) yang memiliki method umum seperti hitungLuas() dan hitungKeliling(). Kemudian, setiap bentuk seperti persegi dan lingkaran akan mengimplementasikan method tersebut dengan cara yang berbeda
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class Bentuk { double hitungLuas(); double hitungKeliling(); void info() { print('Luas: ${hitungLuas()}, Keliling: ${hitungKeliling()}'); } } class Persegi implements Bentuk { double sisi; Persegi(this.sisi); @override double hitungLuas() => sisi * sisi; @override double hitungKeliling() => 4 * sisi; @override void info() { print('Persegi sisi $sisi → Luas ${hitungLuas()} | Keliling ${hitungKeliling()}'); } } class Lingkaran implements Bentuk { double radius; Lingkaran(this.radius); @override double hitungLuas() => 3.14 * radius * radius; @override double hitungKeliling() => 2 * 3.14 * radius; @override void info() { print('Lingkaran radius $radius → Luas ${hitungLuas()} | Keliling ${hitungKeliling()}'); } } void main() { Bentuk b1 = Persegi(4); Bentuk b2 = Lingkaran(7); b1.info(); b2.info(); }

Kesimpulan

Polimorfisme → satu method, banyak perilaku Abstraksi → menyederhanakan sistem Interface & abstract class → membuat struktur program lebih rapi Membuat kode lebih fleksibel, reusable, dan mudah dikembangkan

🔹 Bonus 1: Class Pekerja

Class Pekerja adalah sebuah class (biasanya abstract/interface) yang digunakan sebagai template atau dasar untuk berbagai jenis pekerjaan. Di dalamnya terdapat method umum seperti bekerja() yang harus diimplementasikan oleh class turunan seperti Programmer, Dokter, atau Guru.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class Pekerja { void bekerja(); } class Programmer implements Pekerja { @override void bekerja() { print('Coding aplikasi'); } } class Dokter implements Pekerja { @override void bekerja() { print('Mengobati pasien'); } } class Guru implements Pekerja { @override void bekerja() { print('Mengajar siswa'); } } void main() { List pekerja = [ Programmer(), Dokter(), Guru() ]; for (var p in pekerja) { p.bekerja(); } }

🔹 Bonus 2 Class Table

Class Tablet adalah class yang merepresentasikan perangkat tablet dengan menggabungkan beberapa fungsi, seperti komputer, kamera, dan telepon. Biasanya, class ini dibuat dengan mewarisi (inherit) dari satu class utama (misalnya Komputer) dan menggunakan class lain sebagai tambahan fitur.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A class Komputer { void prosesData() { print('Memproses data...'); } } mixin KameraMixin { void ambilFoto() { print('Tablet mengambil foto'); } } mixin TeleponMixin { void menelepon() { print('Tablet melakukan panggilan'); } } class Tablet extends Komputer with KameraMixin, TeleponMixin { void info() { print('Ini adalah Tablet (Komputer + Kamera + Telepon)'); } } void main() { Tablet t = Tablet(); t.info(); t.prosesData(); t.ambilFoto(); t.menelepon(); }

🔹 Bonus 3 Class Mathutils

Class MathUtils adalah class yang berisi kumpulan fungsi-fungsi matematika (utility) seperti menghitung faktorial, mengecek bilangan prima, menghitung rata-rata, dan lainnya. Biasanya method di dalamnya bersifat static, sehingga bisa digunakan langsung tanpa membuat objek.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A import 'dart:math'; class MathUtils { static final Random _random = Random(); static int faktorial(int n) { if (n < 0) { throw ArgumentError('n tidak boleh negatif'); } if (n <= 1) return 1; return n * faktorial(n - 1); } static bool isPrima(int n) { if (n <= 1) return false; for (int i = 2; i <= sqrt(n).toInt(); i++) { if (n % i == 0) return false; } return true; } static double bulatkan(double angka, int desimal) { double faktor = pow(10, desimal).toDouble(); return (angka * faktor).round() / faktor; } static int randomInt(int min, int max) { if (min > max) { throw ArgumentError('min tidak boleh lebih besar dari max'); } return min + _random.nextInt(max - min + 1); } static double randomDouble(double min, double max) { if (min > max) { throw ArgumentError('min tidak boleh lebih besar dari max'); } return min + _random.nextDouble() * (max - min); } static double rataRata(List data) { if (data.isEmpty) { throw ArgumentError('List tidak boleh kosong'); } double total = data.reduce((a, b) => a + b); return total / data.length; } static double median(List data) { if (data.isEmpty) { throw ArgumentError('List tidak boleh kosong'); } List sorted = List.from(data)..sort(); int n = sorted.length; if (n % 2 == 1) { return sorted[n ~/ 2]; } else { return (sorted[n ~/ 2 - 1] + sorted[n ~/ 2]) / 2; } } } void main() { print(MathUtils.faktorial(5)); print(MathUtils.isPrima(7)); print(MathUtils.bulatkan(3.14159, 2)); print(MathUtils.randomInt(1, 10)); print(MathUtils.randomDouble(1.0, 5.0)); print(MathUtils.rataRata([1, 2, 3, 4, 5])); print(MathUtils.median([1, 2, 3, 4, 5])); }

🔹 Challenge

Challenge Media Player adalah latihan yang bertujuan untuk membuat sistem pemutar media dengan menerapkan konsep abstraksi dan interface. Dalam challenge ini dibuat abstract class MediaPlayer sebagai dasar, serta interface seperti Playable dan Pausable yang berisi fungsi play() dan pause().
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class MediaPlayer { void open(); } abstract class Playable { void play(); } abstract class Pausable { void pause(); } class AudioPlayer extends MediaPlayer implements Playable, Pausable { @override void open() { print("Audio dibuka"); } @override void play() { print("Audio diputar"); } @override void pause() { print("Audio dijeda"); } } class VideoPlayer extends MediaPlayer implements Playable, Pausable { @override void open() { print("Video dibuka"); } @override void play() { print("Video diputar"); } @override void pause() { print("Video dijeda"); } } void main() { AudioPlayer audio = AudioPlayer(); audio.open(); audio.play(); audio.pause(); print(""); VideoPlayer video = VideoPlayer(); video.open(); video.play(); video.pause(); }
